# Getting Started with the PHP AI Client

This guide will help you install and start using the PHP AI Client in your PHP application.

## Requirements

- PHP 8.0 or higher
- Composer for package management

## Installation

Install the PHP AI Client via Composer:

```bash
composer require wordpress/php-ai-client
```

## Current Implementation Status

**⚠️ Important**: The PHP AI Client is currently in active development. This guide covers what's available now and what's coming soon.

### What Works Now ✅

- Provider registry system
- Traditional API methods (`generateTextResult`, `generateImageResult`, etc.)
- Core interfaces and DTOs
- Operation tracking

### What's Coming Soon 🚧

- Fluent API (`AiClient::prompt()` - currently throws RuntimeException)
- Concrete provider implementations (OpenAI, Anthropic, Google)
- Builder classes for easier message construction

## Working with the Current Implementation

### Basic Provider Registration

The provider registry system is fully functional:

```php
<?php

use WordPress\AiClient\AiClient;
use YourNamespace\CustomAIProvider; // Your custom implementation

// Register a custom provider (you'll need to implement this)
AiClient::defaultRegistry()->registerProvider(CustomAIProvider::class);

// Check if provider is registered
if (AiClient::defaultRegistry()->hasProvider('your-provider-id')) {
    echo "Provider registered successfully\n";
}
```

### Traditional API Usage (Available Now)

Use the traditional API methods. The SDK supports flexible input formats:

```php
<?php

use WordPress\AiClient\AiClient;
use WordPress\AiClient\Messages\DTO\UserMessage;
use WordPress\AiClient\Messages\DTO\MessagePart;

// Option 1: Simple string (gets normalized internally)
try {
    $result = AiClient::generateTextResult('Write a haiku about PHP development', $customModel);
    echo $result->toText();
} catch (\RuntimeException $e) {
    echo "Need a registered provider: " . $e->getMessage();
}

// Option 2: Explicit message construction
$message = new UserMessage([new MessagePart('Write a haiku about PHP development')]);
try {
    $result = AiClient::generateTextResult([$message], $customModel);
    echo $result->toText();
} catch (\RuntimeException $e) {
    echo "Need a registered provider: " . $e->getMessage();
}

// Option 3: Without model (auto-discovery - requires registered providers)
try {
    $result = AiClient::generateTextResult('Write a haiku about PHP development');
    echo $result->toText();
} catch (\RuntimeException $e) {
    echo "No suitable models found: " . $e->getMessage();
}
```

### Provider Registry Operations

Discover and work with providers:

```php
<?php

use WordPress\AiClient\AiClient;
use WordPress\AiClient\Providers\Models\DTO\ModelRequirements;
use WordPress\AiClient\Providers\Models\Enums\CapabilityEnum;

$registry = AiClient::defaultRegistry();

// Find providers that support text generation
$requirements = new ModelRequirements([CapabilityEnum::TEXT_GENERATION]);
$providersMetadata = $registry->findModelsMetadataForSupport($requirements);

foreach ($providersMetadata as $providerData) {
    echo "Provider: " . $providerData->getProvider()->getName() . "\n";
    foreach ($providerData->getModels() as $model) {
        echo "  Model: " . $model->getName() . "\n";
    }
}
```

### Operation Tracking

Create and track AI operations:

```php
<?php

use WordPress\AiClient\AiClient;
use WordPress\AiClient\Messages\DTO\UserMessage;
use WordPress\AiClient\Messages\DTO\MessagePart;

// Create an operation for async processing
$messages = [new UserMessage([new MessagePart('Generate a detailed report')])];
$operation = AiClient::generateTextOperation($messages, $customModel);

echo "Operation ID: " . $operation->getId() . "\n";
echo "Operation State: " . $operation->getState()->value . "\n";
```

## Working with Current DTOs

### Creating Messages

Build messages using the available DTO classes:

```php
<?php

use WordPress\AiClient\Messages\DTO\UserMessage;
use WordPress\AiClient\Messages\DTO\ModelMessage;
use WordPress\AiClient\Messages\DTO\SystemMessage;
use WordPress\AiClient\Messages\DTO\MessagePart;

// Create different message types - constructor requires MessagePart array
$userMessage = new UserMessage([new MessagePart('What is PHP?')]);
$systemMessage = new SystemMessage([new MessagePart('You are a helpful programming assistant.')]);
$modelMessage = new ModelMessage([new MessagePart('PHP is a server-side scripting language.')]);

// Messages support multipart content
$multipartMessage = new UserMessage([
    new MessagePart('Here is some text'),
    new MessagePart($fileObject), // File attachment
]);
```

### Working with Results

Handle AI generation results:

```php
<?php

// Assuming you have a working provider implementation
try {
    $result = AiClient::generateTextResult($messages, $customModel);
    
    // Get the generated text
    $text = $result->toText();
    
    // Access token usage information
    $usage = $result->getTokenUsage();
    echo "Tokens used: " . $usage->getTotalTokens() . "\n";
    
    // Get provider-specific metadata
    $metadata = $result->getProviderMetadata();
    print_r($metadata);
    
} catch (\Exception $e) {
    echo "Generation failed: " . $e->getMessage() . "\n";
}
```

### Provider Configuration Checking

Check if providers are properly configured:

```php
<?php

use WordPress\AiClient\AiClient;

$registry = AiClient::defaultRegistry();

// Check if provider is configured (has API keys, etc.)
if ($registry->isProviderConfigured('your-custom-provider')) {
    echo "Provider is ready to use\n";
} else {
    echo "Provider needs configuration\n";
}
```

## Error Handling

Handle potential errors in the current implementation:

```php
<?php

use WordPress\AiClient\AiClient;

try {
    // This will work once you have a provider implementation
    $result = AiClient::generateTextResult($messages, $customModel);
    echo $result->toText();
    
} catch (\RuntimeException $e) {
    // Handle not-yet-implemented features
    echo "Feature not ready: " . $e->getMessage() . "\n";
    
} catch (\InvalidArgumentException $e) {
    // Handle invalid arguments
    echo "Invalid input: " . $e->getMessage() . "\n";
    
} catch (\Exception $e) {
    // Handle other errors
    echo "Error: " . $e->getMessage() . "\n";
}
```
